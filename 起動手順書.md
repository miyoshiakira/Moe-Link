# Moe-Link 起動手順書

## 前提条件

以下のAzureリソースが作成済みであること。

| リソース | 用途 |
|---|---|
| Azure OpenAI Service | GPT-4o Realtime API（音声通話） |
| Azure AI Search | RAGによるキャラ設定・記憶の検索 |

Azure OpenAI に `gpt-4o-realtime-preview` モデルがデプロイ済みであること。

---

## 1. 環境変数の設定（初回のみ）

```bash
cd C:/Git/Moe-Link/backend
cp .env.example .env
```

`.env` を開いて以下の値を設定する。

```env
# Azure Portal > OpenAI リソース > キーとエンドポイント から取得
AZURE_OPENAI_ENDPOINT=https://<your-resource>.openai.azure.com
AZURE_OPENAI_API_KEY=<your-api-key>
AZURE_OPENAI_DEPLOYMENT=gpt-4o-realtime-preview
AZURE_OPENAI_API_VERSION=2024-12-17

# Azure Portal > AI Search リソース > キー から取得（管理者キー）
AZURE_SEARCH_ENDPOINT=https://<your-search>.search.windows.net
AZURE_SEARCH_API_KEY=<your-search-api-key>

PORT=3001
FRONTEND_URL=http://localhost:5173
```

cd C:/Git/Moe-Link/backend

```

```

║   Moe-Link Backend                    ║

**ターミナル2：フロントエンド**

```bash
cd C:/Git/Moe-Link/frontend
npm run dev
```

起動後、ブラウザで http://localhost:5173 を開く。

## 5. 通話の使い方

1. 画面右上の **「通話」ボタン（緑）** をクリック
2. ブラウザからマイクの使用許可を求められたら **「許可」** をクリック
3. 「接続中…」から「通話中」になったら話しかけるだけでOK
   - VAD（音声区間検出）が自動でオン/オフを制御するので、ボタン操作は不要
4. 通話終了は **赤い電話ボタン** をクリック

### 通話中のボタン

| ボタン | 機能 |
|---|---|
| マイクアイコン（左） | ミュート切り替え |
| 赤い電話（中央） | 通話終了・チャット画面に戻る |
| スピーカー（右） | スピーカー/イヤホン表示切り替え（UI表示のみ） |

## 6. トラブルシューティング

**「WebSocket 接続に失敗しました」と表示される**
→ バックエンド（ターミナル1）が起動しているか確認する。

**「環境変数 AZURE_OPENAI_ENDPOINT が設定されていません」と表示される**
→ `.env` ファイルが `backend/` フォルダに存在するか、値が正しく設定されているか確認する。

**シードスクリプトがエラーになる**
→ `AZURE_SEARCH_API_KEY` が管理者キー（クエリキーではなく）であることを確認する。

**音声が聞こえない**
→ ブラウザのマイク許可が「ブロック」になっていないか確認する。Chrome の場合、アドレスバー左の鍵アイコンから変更できる。
